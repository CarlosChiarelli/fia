# FIA: finanças inteligentes e automatizadas

## Objetivo

Prever o comportamento de ações do mercado financeiro e encontrar a carteira (conjunto de ações) com maior retorno possível.

Todas tomadas de decisão serão "data-driven" a partir de aprendizado de máquina, ferramentas e fluxos de ciência de dados.

Será utilizado R e Python neste projeto, pois são recursos gratuitos.

Todos os dados aqui serão do **mercado à vista de ações**.

## Estrutura do projeto

1. Definição do problema

2. Juntando informações (webscraping)

3. Análise exploratória (visualização de dados)

4. Preparação do dados

5. Avaliar modelos preditivos

6. Melhorar precisão dos modelos

7. Finalizar modelo e testar no mercado diário

## 1. Definição do problema

Trata-se de um estudo de mercado financeiro com intuito de ganhar dinheiro ao negociar (comprar/vender) ações do mercado financeiro. Isso será feito coletando dados e modelando as séries temporais que são os preços das ações.

A modelagem preditiva realizada visa prever os preços das ações, sendo implementada com aprendizado de máquina e inteligência artifical, ou seja, ferramentas de ciência de dados.

## 2. Juntando informações (webscraping)

A obtenção dos dados será feita via Internet, ou seja, uma raspagem de dados é necessária para obter os preços das ações com seus respectivos códigos da bolsa de valores de São Paulo.
Toda empresa/ativo/ação/papel possui um código para ser representado na Bovespa, é como se fosse um cpf para uma pessoa.

Para montar o banco de dados do problema basta conseguir esses códigos e depois usar alguma API para obter os preços das ações.

Terá um arquivo .txt aqui com boa parte dos códigos das empresas cadastradas algum dia na bolsa de São Paulo, contudo está desatualizado, podendo haver empresas falidas até faltar empresas cadastradas recentemente.

Por isso será feita uma raspagem desses dados usando Selenium (web-driver) diretamente do site da B3 (bolsa de SP), pois ele está em javascript.

## 3. Análise exploratória (visualização de dados)

Visualização dos dados para conhecer o domínio do problema.

Gerar estatísticas descritivas do tipo e distribuições das variáveis (colunas) explicativas, cruzando elas através de diferentes gráficos que possibilitem enxergar o comportamento da série histórica ao longo do tempo.

## 4. Modelagem preditiva

* Limpeza dos dados
* Seleção de variáveis/"features"
* Transformação de dados e testar suposições

## 5. Avaliar modelos preditivos

* Divisão do conjunto de dados e validação
* Treinar e avaliar diferentes modelos preditivos
* Comparar modelos

## 6. Melhorar precisão dos modelos

* Ajuste de hiperparâmetros do modelos 
* União do modelos treinados (ensembles)
* Salvar o modelo final

## 7. Finalizar modelo e testar no mercado diário

* Montagem uma metologia de trading 
* Elaboração de carteira (conjunto de ações)
* Análise de risco e retorno
